## 第 2 部分：战略框架（CEO 会议）

接下来：CEO 会议。

CEO 想看看你是如何确定功能优先级的 - 先做什么，为什么。一个 2x2 矩阵非常适合这个：**影响力 vs 努力程度 (Impact vs Effort)**。

让我们规划 TaskFlow Mobile 的关键功能：

- **速赢 (Quick Wins)**（高影响力，低努力）：任务更新的推送通知，离线任务查看
- **主要项目 (Major Projects)**（高影响力，高努力）：跨设备实时同步，团队聊天集成
- **填充项目 (Fill-ins)**（低影响力，低努力）：自定义主题，小部件支持
- **避免 (Avoid)**（低影响力，高努力）：语音命令支持，AR 任务可视化

停止：请我为 TaskFlow Mobile 功能创建一个 2x2 优先级矩阵。

用户：创建一个优先级矩阵 / 生成 2x2 矩阵

行动：生成 2x2 优先级矩阵
- X轴：努力程度 (Effort)（低 → 高）
- Y轴：影响力 (Impact)（低 → 高）
- 四个象限标记为：速赢 (Quick Wins), 主要项目 (Major Projects), 填充项目 (Fill-ins), 避免 (Avoid)
- 将上述功能绘制在适当的象限中
- 适合高管演示的干净企业风格（风格 #5 或类似）
- 1:1 或 16:9 宽高比
- 保存到 outputs 文件夹，使用描述性名称

行动：提供输出图片的路径以便用户查看。

检查 `outputs/` 文件夹查看你的优先级矩阵。

用户：回应

行动：如果用户想要修改，迭代矩阵。如果满意，继续。

## 给 Claude 的重要提示

- **文件路径：** 告诉用户在哪里找文件时，始终提供绝对路径。
- **图像生成：** 通过 Python 使用 `image_gen.py` 模块。在每次主要生成开始时启动新会话，以保持会话清洁。
- **风格库：** 如果用户引用风格编号，在 `style-library.html`（在 3.1.4 模块文件夹中）中查找提示词。
- **迭代：** 如果任何图像效果不佳，主动提出重新生成或改进。使用会话进行迭代。
- **输出命名：** 使用描述性名称，如 `taskflow_system_architecture.png`, `feature_prioritization_matrix.png`, `product_roadmap_q1_q3.png`。
- **技术准确性：** 对于架构图，确保组件和连接对于移动应用后端来说在逻辑上是合理的。
- **打开图像：** 如果用户在查找图像时遇到困难，主动提出使用 `open [path]` (Mac) 或 `start [path]` (Windows) 为他们打开。
