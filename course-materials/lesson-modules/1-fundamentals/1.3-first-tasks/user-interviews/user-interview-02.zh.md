# 用户访谈 #2：工程师（IC）画像

**受访者：** David Chen  
**角色：** 高级后端工程师  
**公司：** GrowthLabs（180 人，B 轮）  
**TaskFlow 方案：** Pro（全公司 120 活跃用户）  
**日期：** 2024-10-06  
**访谈者：** 高级 PM  
**时长：** 28 分钟

---

## 背景

David 在 GrowthLabs 2 年，做后端基础设施（Node.js、PostgreSQL）。团队 6 个月前从 Linear 切到 TaskFlow。他是重度用户，每天都用。

---

## 访谈

**Q: 日常如何用 TaskFlow？**

David：“早上第一件事看 TaskFlow。看分配给我的、今天到期的、高优先的。我主要用快捷键，几乎不用鼠标。Cmd+K 搜索，J/K 切换任务。它够快，这很关键。工具慢我就不用。”

**Q: 最喜欢什么？**

David：“速度。真的，秒开无转圈。之前用 Linear 也快，但 TaskFlow 更快。GitHub 集成很棒——PR 关联任务，合并后状态自动更新，少了很多手动。”

**Q: 让你沮丧的点？**

David：“没有暗黑模式！我有时加班，白屏很刺眼。装过浏览器扩展强制暗黑，但很丑。这是我第一需求。还有，移动 Web 很粗糙，我用手机查任务体验不好——滚动多、上下文难看。”

**Q: 获取任务上下文如何？**

David：“上下文超级重要。任务如果只写‘修登录 bug’，我得到处找信息。好 PM 会写详细描述：背景、边界、相关链接。那时 TaskFlow 很好用。Markdown 支持很棒，可以放代码片段、链接。”

**Q: 通知如何？**

David：“唉，太多。我收到一堆邮件：有人评论、分配、关注的任务有风吹草动我都收邮件。我基本关掉了大部分通知，每天自己刷几次 TaskFlow。应该有摘要模式——给我一封汇总邮件，不要 20 封。”

**Q: 团队协作呢？**

David：“评论挺好，@ 提到就能通知，线程也清晰。但很多讨论还是在 Slack 里，讨论完再有人去更新任务。如果 Slack 集成更好就好了，比如在 Slack 里发言能自动同步到关联任务评论。”

**Q: 只能加一个功能，会选什么？**

David：“暗黑模式，毫无疑问。其次也许离线支持？我在飞机上也想至少能查看任务。还有 API 访问——我想写些自动化，比如从错误告警自动建任务。”

**Q: 和 Linear 相比？**

David：“Linear 对工程师的 UX 稍好，快捷键更全，而且它有暗黑模式！但 TaskFlow 更适合跨职能。我们的 PM 和设计都用 TaskFlow，觉得 Linear 太偏工程。所以 TaskFlow 更适合全团队。”

**Q: 还有别的反馈吗？**

David：“搜索可以更强。现在只是文本搜。我想按自定义字段搜，或者组合过滤。比如‘给我看本周到期、指派给我、项目=API、高优先的所有任务’。过滤器能做到一部分，但搜索能更快。”

---

## 关键洞察
- 速度/性能是续用主因
- 暗黑模式为 #1 需求（影响留存）
- 高度重视 GitHub 集成
- 快捷键对重度用户必不可少
- 移动 Web 体验不足
- 通知过载，需要摘要模式
- 任务上下文（详细描述）非常重要
- 希望更好的 Slack 双向集成
- 搜索需更强（支持字段/组合）
- 认可 Markdown 支持技术细节
