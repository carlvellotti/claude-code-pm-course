# 用户访谈 #4：产品设计师画像

**受访者：** Maya Rodriguez  
**角色：** 高级产品设计师  
**公司：** CloudSync（95 人，A 轮）  
**TaskFlow 方案：** Pro（6 人设计团队）  
**日期：** 2024-10-09  
**访谈者：** 高级 PM  
**时长：** 26 分钟

---

## 背景

Maya 负责 CloudSync 核心产品设计，入职 18 个月，6 个月前推动从 Trello 切到 TaskFlow。她管理设计项目并与 PM、工程合作。

---

## 访谈

**Q: 你在设计流程里如何用 TaskFlow？**

Maya：“所有事——设计 Sprint 规划、设计任务跟进、反馈管理。我为每个交付物建任务，附上 Figma 链接，跟进评审周期。它成了我们设计团队的中枢。”

**Q: 为何选 TaskFlow？**

Maya：“Trello 太简单，团队长大后不够用。看了 Linear 觉得太偏工程；Asana 选项太多让人晕。TaskFlow 刚好——够强大但不臃肿。可视化看板对设计师很关键。”

**Q: 哪些做得好？**

Maya：“附件很好——可放 Figma、截图、设计规格。评论用于设计反馈也不错，@ 提醒方便。还有就是快，我不想等加载。”

**Q: 哪些让你烦？**

Maya：“没有暗黑模式要命。我常加班，白屏刺眼。用过浏览器扩展会把 UI 弄坏。还有坐车看设计时，移动体验很糟——细节难看、滚动很多。”

**Q: 与 PM/工程协作？**

Maya：“总体还行。我建设计任务，PM 建特性任务，工程建实现任务，我们互相链接。但偶尔会漏：PM 没拉设计，工程在设计未就绪前就开做。希望能有必经步骤或依赖约束。”

**Q: 处理设计反馈怎么做？**

Maya：“评论能用但会乱。有人反馈，我改了再传新版，线程越拉越长。希望有版本跟踪或更清晰的‘已处理’标记。有时干系人没上下文就留言‘make it pop’，我得去挖他们想要啥。”

**Q: 新设计师入职如何？**

Maya：“很手工。我得带着走流程——项目怎么结构、Figma 链接放哪、怎么打标签。要是有设计项目模板就好了：‘这是一个特性设计周期的预置结构’。我老在重复搭建。”

**Q: 如果只能加一个功能？**

Maya：“暗黑模式，100%。其次是更好的图片/稿件处理。现在截图在任务里很小，想要画廊视图或放大。还有模板——设计项目和反馈格式都想要。”

**Q: 其他反馈？**

Maya：“通知有点多。每条评论、每次分配都 ping。我想只收设计关键事件，或者把工程提问汇成摘要。”

---

## 关键洞察
- 可视化看板对设计流程关键
- 暗黑模式强需求（常晚间工作）
- 移动端查看设计体验不足
- 模板库有用（设计项目结构）
- 通知过载（需要过滤/批量）
- 需要设计版本跟踪/反馈已处理标记
- 希望更好的图片/稿件浏览
- 跨职能协作存在缺口（依赖/必填）
