配色方案：

背景：
- 主背景：#1a1a1a（深灰，非纯黑）
- 次背景：#242424（卡片/面板）
- 三级背景：#2e2e2e（Hover）

文字：
- 主文字：#f0f0f0（偏白，减轻眼疲劳）
- 次文字：#b0b0b0
- 禁用文字：#666666

强调色：
- 品牌蓝：#4d9fff（亮于浅色模式 #2563eb）
- 成功绿：#4ade80
- 警告黄：#fbbf24（与浅色相同）
- 错误红：#f87171
- 链接：#60a5fa（与品牌区分）

边框/分割：
- 细分隔：#333333
- 强分隔：#444444

═══════════════════════════════════════════════════════════════════

关键界面：

1) 项目看板：
- 左侧导航 #1a1a1a
- 主区 #242424
- 任务卡 #2e2e2e，8px 圆角，轻微阴影
- 列头有 #333333 细分隔
- Hover：卡片背景亮到 #363636

2) 任务详情：
- 右侧抽屉
- 背景 #242424，标题 #f0f0f0，大字号
- 描述 Markdown，代码块背景 #1a1a1a
- 评论线程在底部
- 附件缩略图有边框
- 操作按钮用对应强调色（编辑/完成/删除）

3) 顶部导航：
- 固定头部 #1a1a1a
- 搜索框 #2e2e2e，边框 #333333
- 头像/设置图标 #f0f0f0
- 通知红点 #f87171

4) 设置弹窗：
- 遮罩 60% 黑（#000000 @0.6）
- 内容 #242424，圆角 12px
- 表单输入 #2e2e2e
- 暗黑模式开关（开/关/跟随系统）

═══════════════════════════════════════════════════════════════════

重要设计决策：

对比度：
- 文本满足 WCAG AA：主文 (#f0f0f0 on #1a1a1a) 对比 11.5:1；次文 (#b0b0b0) 对比 6.2:1。

不用纯黑：
- “#1a1a1a 比 #000000 更护眼，避免纯白文字在 OLED 上刺眼。”

代码高亮：
- 采用类似 VS Code Dark+ 的暗色主题，示例含 JS/Python/JSON。

用户上传图片：
- 附件加 #333333 边框，防止白底截图突兀。

图标：
- 重要图标 #f0f0f0，次要 #b0b0b0；过细的图标已加粗。

═══════════════════════════════════════════════════════════════════

交互态：

Hover：
- 按钮背景亮约 10%
- 卡片 #2e2e2e → #363636
- 链接出现下划线

Focus：
- 键盘焦点蓝色描边 #4d9fff，2px，4px 偏移

Active：
- 按钮略暗，缩放到 98%

Disabled：
- 不透明度 50%，光标 not-allowed，文字 #666666

═══════════════════════════════════════════════════════════════════

优点：

✓ 对比度优秀，可读性高  
✓ 强调色突出但不刺眼  
✓ 间距与层级一致  
✓ 明暗切换平滑（有动画）  
✓ 保持品牌感（仍像 TaskFlow）  
✓ 代码块观感佳（工程用户友好）

疑虑/问题：

1) 用户上传图片：
- 白底截图会突兀；加边框是否足够？要不要像 Slack 那样加背景纹理？需用真实内容测试。

2) 图表颜色：
- 报表页（velocity、burndown）默认配色可能不适配暗黑；需测试 Recharts 暗黑方案。

3) 三方嵌入：
- Figma/Google Docs 嵌入无法控样式；接受不统一，还是暗黑下隐藏？需决策。

4) 移动端：
- 设计基于 Web。移动是否首发就带暗黑？Alex（移动 PM）需判断。

5) Marketing 站点：
- taskflow.com 是否也做暗黑？还是仅应用？额外 scope。

6) 默认模式：
- 选项：a) 默认亮色；b) 跟随系统（Jordan 推荐）；c) 默认暗黑。需产品/工程定。

技术实现备注：

- CSS 变量：`--background-primary: #1a1a1a; --text-primary: #f0f0f0; ...`，切换主题改 root 变量。
- 持久化：DB 存用户偏好，localStorage 加速加载。
- 动画：0.2s ease，避免闪屏。
- 系统偏好：prefers-color-scheme，尊重 OS 设置。
- 组件例外：图表/嵌入需单独处理，工程识别边缘场景。
