Claude Code for PMs

ccforpms.com
Interactive PM course

1 Install Claude Code

Go to claude.ai/download and run the installer. Open terminal, type claude to sign in.

2 Download Course

Go to ccforpms.com and click Download. Open the folder in Cursor or VS Code.

3 Start Learning

In terminal: claude then /start-1-1 to begin. Claude guides you through each lesson!

Level 1: Foundation

~2 hours

1.1 Welcome

Meet TaskFlow (fictional PM SaaS). Files: @COMPANY.md @PRODUCT.md @PERSONAS.md

1.2 Visualizing Files

Split-screen: editor left, terminal right. See files update live. Use Cursor, VS Code, or Obsidian.

1.3 First Tasks

Type @filename to reference any file. Process notes, analyze research, describe images.

1.4 Agents

"Analyze these 5 files in parallel" ‚Üí Claude spawns 5 workers. Hours of work in minutes.

1.5 Custom Sub-Agents

Create personas in .claude/agents/: Engineer (feasibility), Exec (ROI), UX (usability).

1.6 Project Memory

CLAUDE.md = project constitution. Add goals, constraints, context. Claude reads it every session.

1.7 Navigation

Esc stops task. Esc√ó2 undoes. /clear resets. /compact summarizes long chats.

Level 2: PM Workflows

~1.5 hours

2.1 Write a PRD

"Write a PRD for [feature] using @COMPANY.md @PERSONAS.md @template.md" ‚Äî then ask 3 agents to review before stakeholders see it.

2.2 Analyze Data

Drop CSV/Excel into folder. "Analyze @data.csv ‚Äî what are the top 3 insights for our roadmap?" Claude finds patterns you'd miss.

2.3 Product Strategy

"Compare us to @COMPETITIVE.md ‚Äî where should we invest?" Strategic frameworks + AI = faster decisions.

Key Files to Reference

  • @COMPANY.md ‚Äî mission, values, constraints
  • @PRODUCT.md ‚Äî current features, metrics, goals
  • @PERSONAS.md ‚Äî user types, needs, pain points
  • @COMPETITIVE.md ‚Äî competitor analysis

PRD Pro Tips

  • Always include success metrics ("How do we know this worked?")
  • Add edge cases + error states before eng review
  • Use agent review: "As a skeptical engineer, poke holes in this"

Level 3: Nano Banana

~2 hours

3.1 Image Generation Basics

3.1.1 Setup: Get Google AI Studio API key (~$0.10/image, ~$5 total)
3.1.2 Parameters: aspect ratios (16:9, 1:1, 9:16), resolution, iterations
3.1.3 Golden Rules: be specific, use reference images, describe style
3.1.4 Style database: save winning prompts for brand consistency

3.2 PM Use Cases

3.2.1 User visuals: personas, journey maps, wireframe mockups
3.2.2 Strategy visuals: architecture diagrams, roadmaps, matrices
3.2.3 Launch assets: app store screenshots, social ads, banners
3.2.4 Presentation decks: hero images, icons, data visualizations

Prompt Formula

  • Subject: "A product manager presenting to stakeholders"
  • Style: "flat illustration, modern SaaS aesthetic"
  • Details: "clean white background, purple accent color"
  • Ref: "in the style of @reference.png"

Quick Wins

  • Persona cards: "illustration of [persona], professional headshot style"
  • Feature mockups: "mobile app screen showing [feature], minimal UI"
  • Icons: "simple flat icon for [concept], single color, no background"

Level 4: Vibe Coding

~2 hours

4.1 Setup

Mindset: You're the PM, Claude is the engineer. You describe, Claude codes. Zero coding needed.

4.2 Plan

Claude interviews you: "What pages? What features?" Creates REQUIREMENTS.md. Clear specs = better code.

4.3 Build & Iterate

Shift+Tab = Plan Mode (preview before executing). Test at localhost:3000.

4.4 GitHub

"Google Drive for code" ‚Äî Claude runs git init, commit, push. Your work is backed up.

4.5 Go Live

Deploy to Vercel (free). Get a real URL like yourapp.vercel.app to share with anyone.
