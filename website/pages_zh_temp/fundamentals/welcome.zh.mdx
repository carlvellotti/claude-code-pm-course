## 1.1: 欢迎来到 TaskFlow

 - **完成时间:** 20 分钟
 - **先决条件:** 已安装 Claude Code

> **在 Claude Code 中开始此模块:** 运行 `/start-1-1` 以启动交互式体验。

## ▶️ 关于如何开始的提醒

⚠️ **如果你还没有克隆仓库:** 先回到 [模块 0.2: 启动 Claude Code 并克隆课程](/getting-started/start-and-clone)。你需要计算机上的课程文件才能继续。


**有两种方法在 Claude Code 中启动此课程:**

### 选项一：在文件资源管理器中找到它

导航到你的 Documents 文件夹并找到 `claude-code-pm-course` 文件夹。然后右键点击并选择 "New Terminal at Folder" (在文件夹处新建终端)

它看起来像这样：

![从文件资源管理器打开文件夹](/images/folder-location.png)

然后输入 `claude` 并按 Enter

### 选项二：从终端打开它

在你的终端中输入这些命令：
```bash
cd ~/Documents/claude-code-pm-course
claude
```

这将导航到你的课程文件夹并在其中直接打开 Claude Code。

**无论哪种方式，你都会到达同一个地方。然后，要开始此模块:**

> **在 Claude Code 中开始此模块:** 运行 `/start-1-1` 以启动交互式体验。

这是你开始每个模块的方式！

## 📖 概览

本模块介绍 TaskFlow（你的虚拟公司）并解释课程结构。

**关键要点:** 这是一个动手课程，你将使用 Claude Code 做真实的产品经理工作。


## 🎓 课程如何运作

### 两个部分

**1. 交互式模块 (你现在正在做的)**
- 使用 Claude Code 完成模块
- 每个模块都以斜杠命令开始：`/start-1-1`, `/start-1-2`, 等等。
- Claude 逐步引导你完成练习
- 你实际**做**工作（不仅仅是阅读）
- TaskFlow 的真实产品经理任务

**2. 参考指南 (你现在正在阅读的)**
- 每个模块的综合独立指南
- 可以在网站上阅读，无需使用 Claude Code
- 当你需要更深入的解释时使用它们
- 非常适合复习和稍后参考

### 课程结构

- **模块 1: 基础** - 核心概念和工作流
- **模块 2: 高级** - 真实产品经理工作 (PRD, 数据分析, 战略)

## 🏢 什么是 TaskFlow?

### 公司概览

**TaskFlow** 是为本课程创建的一个虚拟项目管理 SaaS 公司。想象一下 **Asana 遇见 Jira，专为远程优先团队打造。**

**公司详情:**
- **成立时间:** 2021
- **阶段:** B 轮初创公司
- **融资:** 已筹集 $20M
- **团队规模:** 50 名员工 (15 工程, 5 产品, 10 设计, 20 市场/销售)
- **指标:** $2.5M ARR, 10,000 活跃用户, 增长迅速

**你的角色:** 高级产品经理
- 你负责激活和入职
- 向 CEO (Sarah Chen) 汇报
- 与 CTO (Mike Rodriguez) 和设计主管 (Alex Kim) 紧密合作

**使命:** 赋能远程团队无缝协作

**产品:** 结合任务管理、团队协作和远程团队异步优先功能的项目管理工具

**关键差异化因素:** 比 Asana 更简单，比 Linear 更跨职能，专为异步工作打造

### 👥 用户画像

- **Sarah - 企业管理员:** 大型公司的 IT 主管，专注于安全、合规和成本控制

- **Mike - 个人贡献者(IC)工程师:** 软件工程师，专注于快速工具、GitHub 集成和最少干扰

- **Alex - 团队负责人:** 工程经理，专注于团队可见性、工作量平衡和进度跟踪


## 📂 课程文件结构

当你在 TaskFlow 课程中工作时，你会发现以下内容：

```
claude-code-pm-course/
├── company-context/
│   ├── COMPANY.md - 公司概览和你的角色
│   ├── PRODUCT.md - TaskFlow 做什么
│   ├── PERSONAS.md - 3 个用户画像
│   └── COMPETITIVE.md - 竞争格局
├── lesson-modules/
│   ├── 1.1-welcome/ - 你现在在这里！
│   │   ├── CLAUDE.zh.md (Claude 的教学脚本)
│   │   └── REFERENCE_GUIDE.md (此文件)
│   ├── 1.2-visualizing-files/
│   ├── 1.3-reading-files/
│   └── [更多模块...]
├── your-work/
│   └── [你在练习中将创建的文件]
└── CLAUDE.zh.md - 项目记忆 (在 Obsidian 中可见)
```

**注意:** `.claude/` 文件夹（用于命令、智能体、设置）存在但是隐藏的。你将在以后的模块中使用它。

## ⏱️ 时间投入

**每个模块:** 15-30 分钟
**每个等级:** 2-4 小时
**完整课程:** 20-30 小时

**但是:** 你可以按照自己的节奏进行！
- 一次做一个模块
- 在等级之间休息
- 随时回来（进度已保存）

### 你将会看到什么

在整个课程中，你会看到 Claude 运行如下命令：

```
⏺ Read(company-context/COMPANY.md)
  ⎿  Read 45 lines

⏺ Bash(ls -la company-context/)
  ⎿  Listed 4 files
```

**不用担心理解这些！** 它们是 Claude 在幕后工作。你只需专注于产品经理的工作。

### 权限提示

偶尔，你会看到提示询问：
Do you want to allow this command?

**只需点击:** Yes and dont ask again

这让 Claude 能够顺利工作，而无需每次都请求许可。

## ❓ 常见问题

**我需要安装什么吗？**
只需 Obsidian (模块 1.2)，它是免费的。

**我可以跳过模块吗？**
不推荐。每个模块都建立在之前的模块之上。

**完整课程需要多长时间？**
总共 15-25 小时，按你自己的节奏。

**这适合非技术产品经理吗？**
是的！不需要编码。

**我可以将此用于实际工作吗？**
绝对可以。在模块 1 之后，你可以将这些模式应用于你的实际产品经理工作。
